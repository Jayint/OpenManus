## 术语表

- **LLM（大语言模型）**: 指大型语言模型（Large Language Model），用于生成和理解自然语言的深度学习模型。
- **MCP（Model Context Protocol）**: 模型上下文协议，一种用于增强大语言模型与外部工具交互能力的协议。
- **A2A（Agent-to-Agent）协议**: 一种代理间通信协议，支持智能体之间的标准化交互与任务传递。
- **Sandbox（沙箱）**: 一个隔离的运行环境，用于安全地执行代码或操作，防止对主系统造成影响。
- **Agent（智能体）**: 具备自主决策和执行能力的软件实体，能够根据用户指令完成复杂任务。
- **Data Analysis Agent**: 专门用于数据分析和可视化的智能体，集成在 OpenManus 中以处理数据相关任务。
- **VMind**: 由 VisActor 开发的智能图表生成引擎，基于大模型自动生成可视化图表。
- **VChart**: VisActor 提供的可视化渲染库，支持多种图表类型和交互功能。
- **Tool Call**: 指智能体调用外部工具（如浏览器、Python 执行器等）来完成特定任务的行为。
- **Flow**: 多智能体协同工作流，允许不同类型的智能体按顺序或并行执行任务。

## 相关项目与生态系统工具

### OpenManus 生态组件

- **Data Analysis Agent**: 内置于 OpenManus 的数据分析智能体，可通过配置启用，支持数据处理、报告生成和图表可视化。
- **Chart Visualization Tool**: 基于 VMind 和 VChart 的图表生成工具，支持将数据自动转化为可视化图表（PNG 或 HTML 格式）。
- **MCP Server**: 支持 MCP 协议的服务器模块，允许智能体通过标准化接口调用外部工具。
- **A2A 集成模块**: 实现 A2A 协议的适配层，使 OpenManus 能够与其他支持 A2A 的智能体进行交互。

### 外部依赖与工具链

- **Node.js (>=18)**: 用于运行图表可视化工具中的前端依赖。
- **NVM (Node Version Manager)**: 推荐用于管理 Node.js 版本，支持跨平台安装。
- **Playwright**: 可选的浏览器自动化工具，用于支持智能体的网页操作能力。
- **UV**: 推荐使用的 Python 包管理器，提供比 pip 更快的依赖安装速度。
- **Conda**: Python 环境管理工具，可用于创建隔离的运行环境。

### 示例用例

- **日本旅行计划生成**: 示例中展示了如何使用 OpenManus 生成详细的 7 天日本旅行计划，包含行程安排、HTML 手册、地图和文化建议。
  - 输入提示词位于 `examples/use_case/japan-travel-plan/japan_travel_guide_instructions.txt`
  - 输出结果包括 HTML 格式的旅行手册和可视化地图。
